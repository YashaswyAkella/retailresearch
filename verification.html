Equations implemented

Joint-Sales prediction (Eq. 2):

JS = α + β_IDD·IDD + β_IDD²·IDD² + β_Layout·L + β_Layout×IDD·(L·IDD)
    + β_Variety·V + β_Variety×IDD·(V·IDD) + β_Hyper·H + ε

Optimal distance between departments i & j (Eq. 6):

D*ᵢⱼ = −(β_Layout×IDD·L + β_Variety×IDD·V + β_Hyper×IDD·H + β_IDD) / (2·β_IDD²)

Assumptions (documented)

  • Base currency SEK; EUR figures shown in UI use a ~0.087 conversion rate (fixed in i18n.js; change as needed).
  • Random-effects residuals and endogeneity-control residuals are set to 0 when predicting expected values (they average to 0 by construction).
  • For the 3D view, department footprints match Figure 6 averages where available (bread 146.33 m², fish 31.44 m², deli 32.6 m²); others are reasonable grocery defaults (editable live).
  • Default shelf heights are 1.2–2.1 m (typical grocery range). 3D is a future-research extension flagged in the paper [1].
  • The optimizer uses a JavaScript gradient-descent approximation of the BARON global solver used in the paper. For production redesigns, retailer-specific constraints (entrance, loading bays, columns) should be added.
